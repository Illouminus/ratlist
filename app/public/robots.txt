# Two-tier policy:
#
# 1. Social-preview crawlers — allow them onto `/share/<token>` URLs so
#    link previews work in Twitter / Facebook / Telegram / Discord /
#    Slack / iMessage / LinkedIn / WhatsApp. The Vercel Edge Function
#    at `app/api/share/[token].ts` patches per-token `og:image` and
#    `<title>` into the page; without /share/ access these bots see
#    nothing. Listed bots only fetch URLs they're explicitly pointed
#    at (e.g. when a user pastes a link), so opening the door for them
#    doesn't risk leaking user data into a search index.
#
# 2. Search engines + everything else — landing + legal pages only.
#    Everything authed is user-private; share URLs contain tokens that
#    leak who's on the platform if indexed; /login adds no SEO value.
